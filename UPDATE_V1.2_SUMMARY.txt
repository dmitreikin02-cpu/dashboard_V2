✅ ЧТО ДОБАВЛЕНО

1. 📊 ЦЕНТРАЛИЗОВАННЫЕ НОРМАТИВЫ

   Все нормы теперь в одном месте - легко изменить!

   const NORMS = {
       LOGIC_A: {
           receiving: 150,   // MOP Приёмка
           placement: 280,   // MOP Размещение
           picking: 350,     // MOP Сборка
           control: 220      // MOP Контроль
       },
       LOGIC_B: {
           receiving: 120,   // Приёмка
           placement: 250,   // Размещение
           picking: 300      // Сборка
       }
   };

2. 📐 НОВЫЙ LAYOUT ДЛЯ ЛОГИКИ A

   Старое расположение (2x2):          Новое расположение:
   ┌──────┬──────┐                     ┌──────┬──────┐
   │Приём.│Размещ│                     │Приём.│Размещ│
   ├──────┼──────┤         →           ├──────┴──────┤
   │Сборка│      │                     │   Сборка    │
   └──────┴──────┘                     └─────────────┘

   Сборка теперь занимает ВСЮ ШИРИНУ внизу!

3. 🔧 ГИБКАЯ СИСТЕМА LAYOUTS

   • layout-default - для Логики B (стандартная сетка 2x2)
   • layout-custom - для Логики A (Сборка на всю ширину)

📝 КАК ИЗМЕНИТЬ НОРМЫ

1. Откройте dashboard.html
2. Найдите "const NORMS" (строка ~196)
3. Измените нужные числа
4. Сохраните файл
5. Перезапустите start.bat

Подробная инструкция: HOW_TO_CHANGE_NORMS.txt

📊 ПРИМЕРЫ ИЗМЕНЕНИЯ НОРМ

       // ...
       picking: 400,  // Было 350, стало 400
   },

Пример 2: Уменьшить норму Приёмки для Логики B до 100

   LOGIC_B: {
       receiving: 100,  // Было 120, стало 100
       // ...
   }
